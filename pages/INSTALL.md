<h1><a href="#installing-accumulo" aria-hidden="true" class="anchor" id="user-content-installing-accumulo"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installing Accumulo</h1>
<p>This document covers installing Accumulo on single and multi-node environments.
Either <a href="http://accumulo.apache.org/">download</a> or <a href="/apache/accumulo/blob/1.8/README.md#building-">build</a> a binary distribution of Accumulo from
source code.  Unpack as follows.</p>
<pre><code>cd &lt;install location&gt;
tar xzf &lt;some dir&gt;/accumulo-X.Y.Z-bin.tar.gz
cd accumulo-X.Y.Z
</code></pre>
<p>Accumulo has some optional native code that improves its performance and
stability.  Before configuring Accumulo attempt to build this native code
with the following command.</p>
<pre><code>./bin/build_native_library.sh
</code></pre>
<p>If the command fails, its ok to continue with setup and resolve the issue
later.</p>
<h2><a href="#configuring" aria-hidden="true" class="anchor" id="user-content-configuring"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuring</h2>
<p>The Accumulo conf directory needs to be populated with initial config files.
The following script is provided to assist with this.  Run the script and
answer the questions.  When the script ask about memory-map type, choose Native
if the build native script was successful.  Otherwise choose Java.</p>
<pre><code>./bin/bootstrap_config.sh
</code></pre>
<p>The script will prompt for memory usage.   Please note that the footprints are
only for the Accumulo system processes, so ample space should be left for other
processes like hadoop, zookeeper, and the accumulo client code.  If Accumulo
worker processes are swapped out and unresponsive, they may be killed.</p>
<p>After this script runs, the conf directory should be populated and now a few
edits are needed.</p>
<h3><a href="#secret" aria-hidden="true" class="anchor" id="user-content-secret"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Secret</h3>
<p>Accumulo coordination and worker processes can only communicate with each other
if they share the same secret key.  To change the secret key set
<code>instance.secret</code> in <code>conf/accumulo-site.xml</code>.  Changing this secret key from
the default is highly recommended.</p>
<h3><a href="#dependencies" aria-hidden="true" class="anchor" id="user-content-dependencies"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Dependencies</h3>
<p>Accumulo requires running <a href="http://zookeeper.apache.org/">Zookeeper</a> and <a href="http://http://hadoop.apache.org/">HDFS</a> instances.  Also, the
Accumulo binary distribution does not include jars for Zookeeper and Hadoop.
When configuring Accumulo the following information about these dependencies
must be provided.</p>
<ul>
<li><strong>Location of Zookeepers</strong> :  Provide this by setting <code>instance.zookeeper.host</code>
in <code>conf/accumulo-site.xml</code>.</li>
<li><strong>Where to store data</strong> :  Provide this by setting <code>instance.volumes</code> in
<code>conf/accumulo-site.xml</code>.  If your namenode is running at 192.168.1.9:9000
and you want to store data in <code>/accumulo</code> in HDFS, then set
<code>instance.volumes</code> to <code>hdfs://192.168.1.9:9000/accumulo</code>.</li>
<li><strong>Location of Zoookeeper and Hadoop jars</strong> :  Setting <code>ZOOKEEPER_HOME</code> and
<code>HADOOP_PREFIX</code> in <code>conf/accumulo-env.sh</code> will help Accumulo find these
jars.</li>
</ul>
<p>If Accumulo has problems later on finding jars, then run <code>bin/accumulo classpath</code> to print out info about where Accumulo is finding jars.  If the
settings mentioned above are correct, then inspect <code>general.classpaths</code> in
<code>conf/accumulo-site.xml</code>.</p>
<h2><a href="#initialization" aria-hidden="true" class="anchor" id="user-content-initialization"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Initialization</h2>
<p>Accumulo needs to initialize the locations where it stores data in Zookeeper
and HDFS.  The following command will do this.</p>
<pre><code>./bin/accumulo init
</code></pre>
<p>The initialization command will prompt for the following information.</p>
<ul>
<li><strong>Instance name</strong> : This is the name of the Accumulo instance and its
Accumulo clients need to know it inorder to connect.</li>
<li><strong>Root password</strong> : Initialization sets up an initial Accumulo root user and
prompts for its password.  This information will be needed to later connect
to Accumulo.</li>
</ul>
<h2><a href="#multiple-nodes" aria-hidden="true" class="anchor" id="user-content-multiple-nodes"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Multiple Nodes</h2>
<p>Skip this section if running Accumulo on a single node.  Accumulo has
coordinating, monitoring, and worker processes that run on specified nodes in
the cluster.  The following files should be populated with a newline separated
list of node names.  Must change from localhost.</p>
<ul>
<li><code>conf/masters</code> : Accumulo primary coordinating process.  Must specify one
node.  Can specify a few for fault tolerance.</li>
<li><code>conf/gc</code>      : Accumulo garbage collector.  Must specify one node.  Can
specify a few for fault tolerance.</li>
<li><code>conf/monitor</code> : Node where Accumulo monitoring web server is run.</li>
<li><code>conf/slaves</code>  : Accumulo worker processes.   List all of the nodes where
tablet servers should run in this file.</li>
<li><code>conf/tracers</code> : Optional capability. Can specify zero or more nodes.</li>
</ul>
<p>The Accumulo, Hadoop, and Zookeeper software should be present at the same
location on every node.  Also the files in the <code>conf</code> directory must be copied
to every node.  There are many ways to replicate the software and
configuration, two possible tools that can help replicate software and/or
config are <a href="https://code.google.com/p/pdsh/">pdcp</a> and <a href="https://code.google.com/p/parallel-ssh/">prsync</a>.</p>
<h2><a href="#starting-accumulo" aria-hidden="true" class="anchor" id="user-content-starting-accumulo"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Starting Accumulo</h2>
<p>The Accumulo scripts use ssh to start processes on remote nodes.  Before
attempting to start Accumulo, <a href="https://www.google.com/search?q=hadoop+passwordless+ssh&amp;ie=utf-8&amp;oe=utf-8">passwordless ssh</a> must be setup on the
cluster.</p>
<p>After configuring and initializing Accumulo, use the following command to start
it.</p>
<pre><code>./bin/start-all.sh
</code></pre>
<h2><a href="#first-steps" aria-hidden="true" class="anchor" id="user-content-first-steps"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>First steps</h2>
<p>Once the <code>start-all.sh</code> script completes, use the following command to run the
Accumulo shell.</p>
<pre><code>./bin/accumulo shell -u root
</code></pre>
<p>Use your web browser to connect the Accumulo monitor page on port 9995.</p>
<pre><code>http://&lt;hostname in conf/monitor&gt;:9995/
</code></pre>
<p>When finished, use the following command to stop Accumulo.</p>
<pre><code>./bin/stop-all.sh
</code></pre>
